# 第五章 指针与数组

## 5.1 什么是指针

程序中的每个变量和函数都位于内存的某个地方。在 [第 0 章 计算机基础#数据存储](./000-计算机基础.md#03-数据存储) 中我们介绍了在计算机内部，对象需要通过 **地址** 来访问的，而 **指针就是存储对象递增的变量**

> [!tip]
>
> 简单来说，指针就是存储对象地址的变量。计算机内部通常使用一个无符号表示对象的地址。但是地址的取值范围可能不同于整数的范围，所以一定不能用普通整型变量存储地址

众所周知，整数的表示方式与浮点数的表示方式完全不同，数据占用的字节数取决于数据的内容。**要通过指针变量使用它指向的数据，还需要知道数据的类型**

一个不知道指向数据类型的指针是没有任何用处的。因此，指针不仅仅只是某对象的地址，而且还应该关注指向对象的类型

定义指针类型的变量与一般变量类似，但是需要在类型名后面加上一个星号，表示定义指向该类型的指针，而非该类型的变量。

```c
long *pnumber = nullptr;  // c23, c23 之前的标准请使用 NULL
```

变量 `pnumber` 的类型是指向 `long` 类型的指针，写为 `long *`。这个指针变量 `pnumber` 只能存储 `long` 类型的变量的地址。

> [!warning]
>
> 请注意：定义指针时，总是要初始化它。如果还不能为指针提供期望值，就将它初始化为 **空指针**(`nullptr` 或者 `NULL`)

当我们在同一条声明语句中声明指针变量和普通变量时，可能会产生混淆。例如

```c
long * pnumber = nullptr, number = 0;
```

请注意，变量 `number` 是一个 `long` 类型的变量，而不是 `long *` 类型。为了不要产生混淆，请不要在一条声明语句中声明多个变量，而是拆分多条声明语句

```c
long number = 0;
long *pnumber = nullptr;
```

C 语言中，指针指向的对象可以是任何类型，包括自定义类型。例如

```c
double * pvalue = nullptr;
char *pch = nullptr;
```

> [!warning]
>
> 请注意，无论指针指向的对象类型是什么，指针变量本身的大小始终是相同的。指针变量的尺寸仅取决于目标平台的可寻址内存的大小
>

下面的示例程序演示了获取当前机器的指针大小

```c title="ex05_01.c"
/**
 * ex05_01.c - 获取当前平台的指针大小
 * 
**/
#include <stdio.h>

int main(void)
{
    printf("sizeof(double *) = %lu\n", sizeof(double *));
    printf("sizeof(  long *) = %lu\n", sizeof(long *));
    printf("sizeof(  char *) = %lu\n", sizeof(char *));
    return 0;
}
// ex05_01.c
```

目前几乎所有的平台上，指针变量的大小是 $4$ 字节或 $8$ 字节，分别对应 $32$ 位和 $64$ 位计算机架构。

## 5.2 地址运算符

地址运算符 `&` 是一元运算符，用于获取操作数的地址。例如

```c
long number = 12345L;
long *pnumber = &number;
```

表达式 `&number` 返回变量 `number` 的地址。运算符 `&` 可以用于类型 `T` 的变量，返回 `T *` 类型的指针，想要存储这个值就必须使用对应类型的指针变量。

从 C23 开始，我们可以使用 `auto` 关键字自动推导变量的类型。例如

```c
long number = 12345L;
auto pnumber = &number;  // pnumber 的类型是 long *
```

## 5.3 解引用运算符

一旦指针变量指向了对象，就可以使用解引用 `*` 运算符访问存储在对象中的内容

```c
printf("number = %ld; *pnumber = %ld\n", number, *pnumber);
```

之所以称它位解引用运算符是因为访问指针指向内存位置的数据的过程称为 **解除指针的引用**。当使用解引用运算符 `*` 解除指针的引用时，就会得到指针引用对象的别名。也就是说， `*pnumber` 就是变量 `number` 的一个别名，对 `*pnumber` 的任何操作都会作用在 `pnumber` 指向的对象上。下面的示例程序演示了指针的各种用法

```c title="ex05_02.c"
/**
 * ex05_02.c - 指针的各种用法
 * 
**/
#include <stdio.h>

int main(void)
{
    int unit_price = 295;
    int count = 0;
    int discount_threshold = 25;
    double discount = 0.07;

    int *pcount = &count;
    printf("Enter the number of item you want: ");
    scanf("%d%*c", pcount);

    printf("The unit price is $%.2f\n", unit_price / 100.0);

    int *punit_price = &unit_price;
    int price = *pcount * *punit_price;
    int * pprice = &price;

    double net_price = 0;
    double* pnet_price = nullptr;  // C23
    pnet_price = &net_price;
    
    if (*pcount > discount_threshold)
    {
        printf("You qualify for discount of %.0f percent.\n", discount * 100);
        *pnet_price = price * (1 - discount) / 100;
    }
    else 
    {
        net_price = *pprice / 100;
    }
    printf("The net price for %d items if $%.2f\n", *pcount, net_price);
    return 0;
}
// ex05_02.c
```

## 5.4 常量指针和指向常量的指针

当 `const` 关键字用于修饰指针类型时，`const` 关键字位于声明符 `*` 的前面与后面有不同的含义。例如，当 `const` 位于 `*` 前面时

```c
const int * pvalue = nullptr;
```

此时，`const` 用于修饰 `*pvalue`，即指针指向的对象。换句话说，`*pvalue` 是只读的，但是 `pvalue` 不是。此时，指针变量 `pvalue` 称为 **指向常量的指针**。当 `const` 位于 `*` 后面时，

```c
int data = 20;
int * const pdata = &data;
```

此时，`const` 用于修饰 `pdata`，即指针本身。此时，指针变量 `pdata` 是只读变量，但是 `*pdata` 不是。此时，指针变量 `pdata` 称为 **常量指针**。

如果希望某个指针变量指向一个常量，并且指针变量不能指向其他的对象。这样的指针称为 **指向常量的常量指针**，可以在声明符 `*` 的前后都假设关键字 `const`

```c
const float value = 3.14159f;
const float *const pvalue = &value;
```

## 5.5 指向数组的指针

前面我们演示的指针都是指向一个标量的对象，当然，**指针也可以指向一个数组对象**。前面我们说过，**数组的长度是数组类型的一部分**，当声明指向数组的指针时，也需要说明数组元素的类型和数组的长度。也就是说，如果一个数组的类型是 `T[N]`，则指向该数组的指针类型就是 `T(*)[N]`

下面示例代码演示了如何声明并初始化一个指向数组的指针

```c
#include <stdio.h>

int main(void)
{
    int array[10] = {};  // c23
    int (*parray)[10] = &array;  // parray 表示一个指针，指向元素类型为 int 长度为 10 的数组 
    
    for (int i = 0; i < 10; ++i)
    {
        // 使用指向数组的指针修改数组中的元素
        (*parray)[i] = i + 1;  // 先解除引用，然后在进行索引
    }

    for (int i = 0; i < 10; ++i)
    {
        // 使用 array 数组名访问元素
        printf("%d ", array[i]);
    }
    printf("\n");
    
    return 0;
}
```

指向数组的指针在 C 语言中用处不大，只有在通过指针访问多维数组元素时才会用到。这样我们就可以将指向一维数组的指针理解为指向二维数组元素的指针。

## 5.6 指针与数组

指针与数组名之间有很密切的关联。实际上，**在大多数情况下，可以把数组名当作指针**。因此，我们可以直接使用数组名初始化一个指针

```c
double values[10];
double *pvalue = values;  // 编译器进行隐式类型转换
```

语句 `double *pvalue = vlaues;` 使用数组 `values` 的地址初始化指针变量 `pvalue`。

> [!warning]
>
> 请注意，尽管数组名被编译器解释为指针，但是它并不是指针。它本质上还是 `T[N]` 类型的变量。指针变量可以随时指向其他对象，但是 **数组名表示的地址是固定的**，不允许修改。

### 5.6.1 指针的算术运算

在 C 语言中，指针支持算术运算，但是 **只能对指针进行加上或减去一个整数** 或者 **两个指向同一个数组中元素的指针进行减法运算**。其余算术运算作用于指针会导致未定义行为

给指针加上一个整数，其结果是一个地址。也可以从一个指针减去一个整数，其结果也是一个地址。例如，使用下面的语句对指针加 $1$

```c
++pvalue;
```

然而，存储在指针变量中的地址并不是按照算术运算那样递增 $1$，给指针加 $1$ 表示给他递增一个所指向类型的元素。编译器知道存储一个数组元素需要的字节数，给指针加 $1$ 就是给指针中存储的地址增加该字节数。换句话说，**给指针加 $1$ 就是把指针移向数组中的下一个元素**。例如，如果 `pvalue` 是指向 `double` 的指针，`double` 类型占 $8$ 字节，这样 `pvalue` 中的地址就增加 $8$。下图演示了这个过程

![1773151682237.png](https://fastly.jsdelivr.net/gh/duyupeng36/images@master/2026-03/1773151682261.png)

> [!tip]
>
> 数组名被编译器转换为指针时，表示的是数组首元素的递增，即 `&values[0]`
>

当然，`pvalue` 不一定指向 `values` 数组的开始。下面的语句把 `values` 数组中的第三元素的地址赋予给该指针

```c
pvalue = &values[2];
```

现在，`pvalue + 1` 就指向了数组元素 `values[3]`。一般情况下，`pvalue + n` 的结果是 `pvalue + n * sizeof(T)`。这个逻辑也可以应用在从指针减去一个整数。换言之，**递增或递减指针的操作将根据指针所指向的对象的类型来执行**

当然，也可以对执行了算术运算的指针进行解引用运算符，否则对指针的算术运算将毫无意义。假设 `pvalue = &values[3]`，则

```c
*(pvalue + 1) = *(pvalue + 2);
```

等价于

```c
values[4] = values[5];
```

表达式 `pvalue + 1` 并不会修改 `pvalue` 的值，这与表达式 `++pvalue` 是不同的。在递增或递减指针包含的递增后，要解除对指针的引用，就需要使用圆括号。因为，解引用运算符 `*` 的优先级高于算术运算符 `+` 或 `-`。表达式 `*pvalue + 1` 是对 `pvalue` 执行对象的值加 $1$，也就是 `values[4] + 1`。

> [!warning]
>
> 有些书籍会解释 `*pvalue++` 和 `*++pvalue` 等表达式的区别。我们没必要这么做。在需要的时候添加圆括号会使得表达式更清晰
> 
> 当然，理解了运算符的优先级这些表达式也是非常简单的

---
<div style="font-size:20px;font-weight:bold; text-align: center;">计算两个指针之间的差</div>

可以从一个指针中减去另一个指针，但是要求这个两个指针必须指向同一个数组中的元素。否则，这样的运算是没有任何意义的，并且 C 编译器并不会有特殊提示

假定有一个一维数组 `numbers`，其类型是 `long`，定义语句如下

```c
long numbers[] = {10, 20, 30, 40, 50, 60, 70, 80};
```

定义并初始化两个指针变量

```c
long *pnum1 = &numbers[6];  // 指向第 7 个元素
long *pnum2 = &numbers[1];  // 指向第 2 个元素。相差 5 个元素
```

现在，计算这个两个指针之间的差

```c
auto difference = pnum1 - pnum2;  // 结果是 5
```

变量 `difference` 被设置为整数 $5$，因为指针之差由元素决定，而不是由字节数决定。下图演示了

![1773153293575.png](https://fastly.jsdelivr.net/gh/duyupeng36/images@master/2026-03/1773153293587.png)

现在，还有一个问题，`difference` 变量的类型是什么？显然，应该是一个有符号整数类型，以便于能够处理下面的语句

```c
auto difference2 = pnum2 - pnum1;
```

指针变量的尺寸通常由平台决定，一般情况下可能是 $4$ 字节，也可能是 $8$ 字节。存储指针偏移需要的字节数在不同平台可能是不相同的。因此，C 标准规定，两个指针相减，得到的值的类型是 `ptrdiff_t` 类型，这是定义在 `<stddef.h>` 头文件中的类型，本质上就是一个有符号整数类型平台特定的类型别名。通常是 `long` 或者 `long long` 类型

### 5.6.2 指针的比较运算

在 [第三章 决策#比较运算符](003-决策.md#31-比较运算符) 中介绍的比较运算符都可以安全的用在比较两个指向同一个数组中元素的指针。本质上，**是比较指针存储的地址的大小**，如果两个指针不是指向同一个数组的，那么比较起来就没有任何意义

```c title="ex05_03.c"
/**
 * ex05_03.c - 指针算术运算与比较运算
 * 
**/
#include <stdio.h>

int main(void)
{

    int height[] = {176, 168, 190, 182, 175, 155};
    const size_t size = sizeof(height) / sizeof(height[0]);
    int total = 0;

    for (int * p = height; p < height + size; ++p)
    {
        total += *p;
    }

    printf("total = %d\n", total);
    return 0;
}
// ex05_03.c
```

### 5.6.3 使用数组名的指针表示

可以把数组名当作指针来确定数组元素的地址。例如

```c
long data[5] = {};
```

可以使用指针表示法把元素 `data[3]` 表示为 `*(data + 3)`。这种表示法可以应用于一般情形，如元素 `data[0]` 可以表示为 `*(data + 0)`，元素 `data[1]` 表示为 `*(data + 1)`。数组名 `data` 本身表示数组开始时的地址，即首元素的地址；而表达式 `data + 2` 表示偏离起始位置 $2$ 个元素的地址

下面的示例程序演示了这种用法。这个示例程序用于计算前 $100$ 个素数

```c title="ex05_04.c"
/**
 * ex05_04.c - 计算前 100 个素数
 *
 * 一个合数一定可以分解为几个素数的乘积，也就是说，一个数如果能被一个素数整除就一定是合数。
**/
#include <stdio.h>

int main(void)
{
    constexpr size_t size = 100;    // 素数表的长度
    long primes[size] = {[0]=2};    // 素数表
    size_t count = 1;               // 当前素数表中元素的个数
    long trial = 3L;                // 候选的素数

    while (count < size) {
        // 假设候选素数是素数
        bool is_prime = true;
        for (size_t i = 0; i < count && is_prime; ++i) {
            is_prime = trial % *(primes + i) != 0;  // 所有的素数都不是候选素数的因数
        }
        if (is_prime) {
            *(primes + count) = trial;
            ++count;
        }
        trial += 2; // 下一个候选素数
    }
    // 输出一个表格
    printf("The first %lu primes are: \n", size);
    for (size_t i = 0; i < size; ++i) {
        printf("%7ld", *(primes + i));
        if ((i + 1) % 10 == 0) {
            putchar('\n');
        }
    }
    putchar('\n');
    return 0;
}
// ex05_04.c
```

## 5.7 指针与多维数组

指针与数组介绍就是指针指向数组的元素，并通过指针算术运算让指针在数组上进行滑动。一旦这样理解，我们就可以很容弄明白指针与多维数组的关系。我们以二维数组为例

二维数组本质上就是一维数组的一维数组。当使用指针指向一个一维数组时，就需要声明这个数组元素的类型以及数组的长度。例如

```c
double carrots[3][4] = {
    {2.5, 3.2, 3.7, 4.1},
    {4.1, 3.9, 1.6, 3.5},
    {2.8, 2.3, 0.9, 1.1}
};
```

二维数组的元素是 `double[4]` 类型的一维数组。指向该类型数组的指针应该声明为

```c
double (*p)[4];
```

> [!warning]
>
> 注意，在这个声明语句中，`*p` 两边的圆括号是不能省略的。如果省略，即 `double *p[4]` 表示得将不在是指针，而是一个数组，数组元素的类型是 `double *`。这种类型将在字符串章节中介绍
>

下面的示例程序演示了如何使用指针访问二维数组元素的

```c title="ex05_05.c"
/**
 * ex05_05.c - 通过指针访问二维数组
 * 
**/
#include <stdio.h>

int main(void)
{
    double carrots[3][4] = {
        {2.5, 3.2, 3.7, 4.1},
        {4.1, 3.9, 1.6, 3.5},
        {2.8, 2.3, 0.9, 1.1}
    };

    for (double (*p)[4] = carrots; p < carrots + 3; ++p) {
        for (double *q = *p; q < (*p) + 4; ++q) {
            printf("%7.2f", *q);
        }
        printf("\n");
    }
    return 0;
}
// ex05_05.c
```
