# 快速入门

学习 C 语言编程的最佳方法就是亲自动手写代码，而传统的入门程序就是在屏幕上输出 `"hello, world!"`。下面给出了示例程序 [hello.c](../src/ch01/hello.c) 的完整代码

```c title="hello.c"
/* hello.c - 向终端输出 hello, world */

#include <stdio.h>
#include <stdlib.h>

int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {
    //
    printf("hello, world!\n");
    return EXIT_SUCCESS;
}
```

> [!tip]
>
> 注意，我们将示例程序保存为 `hello.c` 文件，文件扩展名 `.c` 表示该文件中包含的是 C 语言源代码
>

稍后我们详细介绍这段代码的每一行内容，现在我们来看如何编译并运行它。要编译这个程序需要一个编译器。针对不同的操作系统，可以有不同的选择，下表列出了常用操作系统上的可选编译器或IDE

| Linux   | [GCC](https://gcc.gnu.org/) [Clang+LLVM](https://clang.llvm.org/)  [qt creator](https://www.qt.io/development/tools/qt-creator-ide)                                                      |

`gcc` 就是我们安装的编译器名字。选项 `-Wall` 告诉编译器为我们提供任何它认为所有不寻常的告警。选项 `-o hello` 告诉编译器将输出存储在名为 `hello` 的文件中。`hello.c` 是 C 源代码文件的名字。现在，来逐行解释 [`hello.c`](../src/ch01/hello.c) 程序

示例程序  [`hello.c`](../src/ch01/hello.c) 的第一行 **注释**：包围在 `/* .... */` 之内的内容会被编译器忽略，这是解释代码和记录程序思路的好地方

```c
/* hello.c - 向终端输出 hello, world */
```

随后的两行都是 `#include <....>` 用于告诉编译器在本程序中包含 `<stdio.h>` 和 `<stdlib.h>`。`<....>` 一般只在此处使用，表示在标准系统目录中查找。标准系统目录取决于目标系统、GCC 的配置方式以及安装位置

```c
#include <stdio.h>
#include <stdlib.h>
```

在我使用的系统上，`#include <....>` 会在下面的标准目录中查找。

```shell
End of search list.
....
```

在随后的几行就是示例程序 [`hello.c`](../src/ch01/hello.c) 要执行的主要内容。下面的代码是`main` **函数定义**

```c
int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {
    ....
}
```

我们将 `main` 函数定义为返回 `int` 类型的值；圆括号 `(int argc, [[maybe_unused]] char* argv[argc + 1])` 表示 `main` 函数调用时需要提供的参数类型和数量；花括号 `{....}` 包围的内容就是 `main` 函数调用时需要执行的语句

> [!tip]
>
> `main` 函数是 C 程序启动时调用的函数。当从命令行或其他程序中调用该程序时，`main` 函数定义了在 **托管环境** 中的主入口
>
> C 语言定义了两种可能的执行环境：**独立环境** 和 **托管环境**。一般而言，**独立环境不提供操作系统**，通常用于嵌入式编程。这类实现提供了最小数量的库函数，**程序启动时调用函数的名称和类型由实现定义**。托管环境就是我们平时编程使用最多的环境，它提供操作系统和全量的库函数

`main` 函数主要执行两条语句。首先，调用 `printf` 函数，这个函数来在 `<stdio.h>` 头文件中声明，该函数用于 **格式化输出** 字符串到屏幕。

```c
printf("hello, world!\n");
```

由 `"...."` 包围的起来的字符序列称为 **字符串字面值**，例如 `"hello, world!\n"` 就是一个字符串字面值。其中字符序列 `\n` 被称为 **转义字符序列**，用于表示换行符。打印时如果遇见 `\n`，输出打印将换行，从下一行的左端行首位置重新开始打印

> [!warning]
>
> 请注意，`"...."` 包围起来的字符串中不能通过键盘上的 `Enter` 键输入换行。
>
> ```c
> printf("hello, world!
> ");
> ```
>
> 这样在编译时 C 编译器将产生一条错误信息。
>
> 此外，`printf` 函数永远不会自动换行，因此我们可以将一条非常长的字符串分阶段输出
>
> ```c
> printf("hello");
> printf(", ");
> printf("world!");
> printf("\n");
> ```

转义字符序列为表示无法输入的字符或不可见 字符提供了一种通用的可扩充的机制。C 标准提供的转移字符序列包括下表列出的几个

| `\'`   | 单引号 `'` | `0x27`        |
| `\"`   | 双引号 `"` | `0x22`        |

在示例程序 [`hello.c`](../src/ch01/hello.c) 中的`main` 函数最后执行 `return` 语句返回一个整数值，此时 `main` 函数结束。

```c
return EXIT_SUCCESS;
```

> [!tip]
>
> `main` 函数返回后就意味着整个 C 程序执行完毕。`main` 函数的返回值通常用于指示程序是否正常结束。如果程序正常结束返回 `EXIT_SUCCESS`。它是一个定义在 `<stdlib.h>` 中的对象式宏，通常会扩展为 $0$

## 1 变量与算术表达式

下面的示例程序 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 使用公式 $\degree C = \frac{5}{9} \times (\degree F - 32)$ 打印华氏度与摄氏度的对照表

```c title="fahr2cels.c"
/* fahr2cels.c - 计算华氏温度和摄氏温度对照表 */

#include <stdio.h>
#include <stdlib.h>

int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {

    int lower = 0; // 华氏温度下限
    int upper = 300;// 华氏温度上限
    int step = 20;// 步长

    int fahr = lower;
    while(fahr <= upper) {
        int cels = 5 * (fahr - 32) / 9;
        printf("%d\t%d\n", fahr, cels);
        fahr = fahr + step;
    }
    return EXIT_SUCCESS;
}
```

这个示例程序引入了一些新概念，包括 **标识符** **声明** **变量** **算术表达式** **循环** 和 **格式化输出**。我们先编译并运行 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 程序，然后在简单介绍这些基本概念。

### 1.1 声明

在 C 语言中，使用任何 **标识符** 之前都必须先 **声明**。声明用于指定该标识符应该表示什么。换句话说，**声明就是指定标识符代表数据的类型**。我们在示例程序 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 中使用的标识符都进行了有效的声明，如果它们作为 **"仅声明"** 看起是这样的

```c
int main(int, char*);
int argc;
[[maybe_unused]] char* argv[];
int lower;
int upper;
int step;
int fahr;
int cels;
```

> [!tip]
> 
> 这些声明都遵循同一种模式。每个声明都有一个 **标识符**，以及与该标识符相关的 **属性** 说明
> 
> + `argc` 的类型是 `int`
> + `main` 后面跟随了一个括号`(....)`，因此声明了一个返回值为 `int` 类型的函数
> + `argv` 后面跟随了一个方括号 `[...]`，因此声明了一个数组，元素类型是 `char*`。属性 `[[maybe_unused]]` 表明它可能未被使用

**标识符** 就是程序员或者 C 标准赋予程序中某些实体的 **名字**。标识符在程序中可以扮演不同的角色

| 角色       | 描述                              |
| :------- | :------------------------------ |
| **对象**   | 这些也称为变量                         |
| **类型别名** | 指定对象的类型。通常以 `_t` 结束，例如 `size_t` |
| **函数**   | 例如 `main` `printf`              |
| **常量**   | 例如 `EXIT_SUCCESS`               |

> [!tip]
> 
> **对象** 是可以在其中表示值的一块存储区域。C 标准将对象定义为 **执行环境中的数据存储区域，其内容可用于表示值**。此外，对象在被引用时，对象可以具有特定的类型。在 C 语言中，对象通常就是指变量

数据对象和函数的声明仅仅知识描述标识符的属性，并不会创建它。因此重复声明不会造成太大损害，但会增加冗余。然而，如果在 **同一个作用域** 中对同一标识符存在多个相互矛盾的声明时，标识符

> [!warning]
> 
> 作用域：**程序中标识符可见的部分**。标识符的作用域绑定在它们的声明出现的作用域内。例如，本例中，标识符 `lower` 和 `upper` 的作用域都是在 `main` 函数的依赖块 `{....}` 内部

标识符的作用域由 C 语言的语法规则明确地描述。下表列出了 C 语言中由语法规则确定的作用域类型

| 作用域         | 声明位置        | 可见范围            | 典型例子      |
| ----------- | ----------- | --------------- | --------- |
| **文件作用域**   | 所有函数之外      | 声明处 → 文件末尾      | 全局变量、顶层函数 |
| **块作用域**    | `{....}` 内部 | 声明处 → 所在块末尾 `}` | 局部变量、函数形参 |
| **函数原型作用域** | 函数原型 `()` 内 | 仅在 `()` 内       | 原型中的形参名   |
| **函数作用域**   | 函数体内（标签）    | 整个函数体           | `goto` 标签 |

在示例程序 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 中，变量的声明都是使用的类型 `int`，它表示其后所列的变量都是整数，与之对应的，`float` 表示其后所列变量都是 **浮点数**(带有小数部分的数)。这两个类型的取值范围取决于具体实现和编译平台。目前，我们只需要知道 `int` 类型一般占 $16$ 位或 $32$ 位；`double` 类型通常占 $64$ 位即可。

### 1.2 定义

通常，声明只指定标识符所引用的对象的类型，而不指定标识符的具体值，也不指定标识符所引用的对象的位置。这个重要的角色是由 **定义** 来完成的

> [!tip]
> 
> 声明指定标识符，而定义指定对象。

一般而言，变量都应该进行 **初始化**，它增加了一个声明，并为对象提供了一个初始值

```c
int lower = 0; // 华氏温度下限
int upper = 300;// 华氏温度上限
int step = 20;// 步长
```

上述三条语句是标识符的声明，同时为对象提供了一个初始值。在 C 语言中，这种带有初始化的声明也用相应的名称定义对象：也就是说，它指示编译器提供可以存储变量值的内存块

对于一个函数，如果它的声明后面跟着一个包含函数代码的大括号 `{....}`，我们就有定义了一个函数

```c
int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {
    ....
}
```

在我们的示例程序 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 中，我们已经看到了两个不同角色的标识符：**对象** 和 **函数**。与对象或函数声明不同的是，**对象或函数的定义必须是唯一的**

> [!warning]
> 
> 对于一个可执行的 C 程序，任何对象和函数有且只有一个定义。否则，执行时将不知道去哪里寻找它们；即使能够执行，执行也可能变得不一致

### 1.3 语句

示例程序 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 中 `main` 函数的第二部分主要由语句组成。例如

```c
while(fahr <= upper) {
    int cels = 5 * (fahr - 32) / 9;
    printf("%d\t%d\n", fahr, cels);
    fahr = fahr + step;
}
```

> [!tip]
> 
> **语句** 是告诉编译器如何处理已经声明的标识符的指令

`while` 语句是 C 语言中提供的最简单的 **循环语句**，其执行过程为：首先，计算圆括号内表达式的值(本例，为 `fahr <= upper`)；如果表达式的值是 `true`，则执行 `while` 语句的依赖块 `{....}`，然后重新计算圆括号内表达式的值；如果表达式的值是 `false`，则结束循环。

> [!tip]
> 
> `true` 和 `false` 是 C23 标准引入的 `bool` 类型的字面值。如果需要在 C23 标准之前使用，则需要将 `<stdbool.h>` 包含到当前文件中。
> 

示例程序 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 的大部分工作都在 `while` 语句的依赖块中完成。语句 

```c
int cels = 5 * (fahr - 32) / 9;
```

使用表达式 `5 * (fahr - 32) / 9` 的值初始化变量 `cels`。在该语句中，我们把表达式写成先乘以 $5$ 然后在除以 $9$ 而不是直接写成 `5 / 9`，这是因为在 C 语言中，整数除法会将结果的小数部分截断，结果中的任何小数部分都会舍弃。由于 $5$ 和 $9$ 都是整数，`5 / 9` 经过截断后的结果为 $0$

下面的语句是函数调用语句

```c
printf("%d\t%d\n", fahr, cels);
```

第一个参数称为 **格式字符串**，包含两个 **格式说明符**。它们表示输出中要插入数字的位置。格式字符串中的每个格式说明符按顺序分别对应 `printf` 函数的剩余参数。格式说明符以 `%` 字符开头。第二个参数 `fahr` 代表的内容将被打印到第一个格式说明符 `%d` 的位置。第三参数 `cels` 代表的内容将被打印到第二个格式说明符 `%d` 的位置。

在 C 语言中的格式说明符还可以指定其他内容用于美化输出的样式。这里我们先介绍 **最小宽度**。例如

```c 
printf("%3d%6d\n", fahr, cels);
```

打印 `fahr` 和 `cels` 时，`fahr` 的值将占 $3$ 字符宽度，不足三个字符的数值使用空格在左边补齐；`cels` 的值将占 $6$ 个字符宽度，同样使用空格在左边补齐。这样输出的结果为

```shell hl:3-8

➜ ./fahr2cels
```

### 1.4 double 类型

还有一个非常重要的问题，由于我们使用的是整型算术运算，因此，经计算得到的摄氏温度值不太精确。例如，$0\degree F$ 对应的精确摄氏温度为 $-17.8\degree C$，而不是 $-17 \degree C$。为了修复这个 BUG 我们应该该用 `double` 类型。下面的示例程序 [`fahr2cels_double.c`](../src/ch01/fahr2cels-double.c) 是 [`fahr2cels.c`](../src/ch01/fahr2cels.c) 的 `double` 版本

```c title="fahr2cels_double.c"
/* fahr2cels_double.c - 计算华氏温度和摄氏温度对照表 */

#include <stdio.h>
#include <stdlib.h>

int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {

    double lower = 0.; // 华氏温度下限
    double upper = 300.;// 华氏温度上限
    double step = 20.;// 步长

    double fahr = lower;
    while(fahr <= upper) {
        double cels = 5 * (fahr - 32) / 9;
        printf("%6.2f%10.2f\n", fahr, cels);
        fahr = fahr + step;
    }
    return EXIT_SUCCESS;
}

```

## 2 for 语句

对于同一个任务，我们可以是采用多种方法实现。下面的示例程序 [`fahr2cels_for.c`](../src/ch01/fahr2cels-for.c) 采用 `for` 语句实现温度转换程序

```c title="fahr2cels_for.c"
/* fahr2cels_for.c - 计算华氏温度和摄氏温度对照表 */

#include <stdio.h>
#include <stdlib.h>

int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {

    for (int fahr = 0; fahr <= 300; fahr = fahr + 20) {
        printf("%3d%10.2f\n", fahr, (5.0 / 9.0) * (fahr - 32));
    }
    return EXIT_SUCCESS;
}
```

示例程序 [`fahr2cels_for.c`](../src/ch01/fahr2cels-for.c) 与示例程序 [`fahr2cels_double.c`](../src/ch01/fahr2cels-double.c) 相比，它们执行结果相同，但程序本身不同。主要的修改在于它去掉了大部分变量，而只使用了一个 `int` 类型的变量 `fahr`。温度表的上限、下限以及步长使用的都是字面量。并且，计算摄氏温度的表达式现在变成了 `printf` 函数的第三个参数

> [!tip]
> 
> 这是示例程序告诉我们，在 C 语言中，运行使用某种类型变量值的任何场所，都可以使用该类型的更复杂的表达式
> 

`for` 语句也是一种循环语句，它是对 `while` 语句的推广；它由 $4$ 部分组成。跟随在 `for (....)` 的由花括号包围的 **块**(这里是由`{....}`标记的 **复合语句**) 称为 `for` 语句的 **依赖块**，它是要重复执行的代码，也称为 **循环体**。其他三部分是在 `(....)` 内部，被分号分隔。第一部分 

```c
int fahr = 0;
```

是循环变量 `fahr` 的声明、定义或初始化，**只在整个 `for` 语句的其他部分之前执行一次**。第二部分

```c
fhar <= 300
```

是 `for` 语句的控制表达式。循环控制将对控制表达式求值，如果结果为真(`true`)，则执行循环体。此后，将执行第三部分

```c
fahr = fahr + 20;
```

将循环变量 `fahr` 增加一个步长。然后，循环控制再次对控制表达式求值。如果计算结果为假(`false`)，循环将终止执行。

> [!tip]
> 
> 位于 `(....)` 内部的三个部分可以是任何表达式。第一部分还可以循环变量的声明或定义。然而，我们希望在 `for` 循环的上下文中严格定义循环变量，因此，以在大多数情况下，第一部分应该是一个变量定义

示例程序 [`fahr2cels_reverse.c`](../src/ch01/fahr2cels-reverse.c) 以逆序（华氏温度按照 $300$ 到 $0$ 的顺序打印）打印温度转换表

```c title="fahr2cels_reverse.c"
/* fahr2cels_reverse.c - 计算华氏温度和摄氏温度对照表 */

#include <stdio.h>
#include <stdlib.h>

int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {

    for (int fahr = 300; fahr >= 0; fahr = fahr - 20) {
        printf("%3d%10.2f\n", fahr, (5.0 / 9.0) * (fahr - 32));
    }
    return EXIT_SUCCESS;
}
```

## 3 常量

在温度转换的示例程序中，直接使用了 $300$ $20$ 等类似的字面值，这些字面值被称为 **幻数**，这不是一个好习惯，它们几乎无法向阅读该程序的人提供任何信息，而且会极大增加程序修改的难度

> [!warning] 
> 
> 尽量避免在程序中使用幻数。一种有效避免的方法就是赋予它们有意义的名字

从 23 标准起，有两种方式赋予幻数有意义的名字：**符号常量** 和 **constexpr**

### 3.1 符号常量

在 C 语言中，可以使用预处理指令 `#define` 把符号名定义为一个特定的替换文本

```c
#define 名字 替换文本
```

在 `#define` 指令之后，程序出现的所有在 `#define` 中定义的名字(没有被引号引起来，也不是其他名字的一部分)都将使用相应的文本进行 **简单替换**

> [!tip] 
> 
> 替换文本可以是任何序列，而不仅限于数字

下面的示例程序 [`symbolic-constants.c`](../src/ch01/symbolic-constants.c) 演示了符号常量的使用方式

```c title="symbolic-constants.c"
/* symbolic-constants.c - #define NAME REPLACE-TEXT */

#include <stdio.h>
#include <stdlib.h>

#define LOWER 0
#define UPPER 300
#define STEP 20


int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {

    for (int fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP) {
        printf("%3d%10.2f\n", fahr, (5.0 / 9.0) * (fahr - 32));
    }
    return EXIT_SUCCESS;
}
```

其中 `LOWER` `UPPER` 和 `STEP` 都是符号常量，而非变量，因此不需要出现在声明。符号常量名通常采用全大写字母

> [!warning] 
> 
> 请注意：`#define` 指令行的末尾没有分号，也不能分号

## 3.2 constexpr

符号常量的问题就是无法为程序阅读人员提供类型信息。为了解决这个问题，C23 标准引入了 `constexpr` 关键字。

`constexpr` 关键字用于修饰对象的定义，**指示编译器在编译时计算对象的值**。此外，被 `constexpr` 修饰的对象只允许使用 **常量表达式** 进行初始化。下面的示例程序 [`constexpr.c`](../src/ch01/constexpr.c) 演示了 `constexpr` 关键字的使用

```c title="constexpr.c"
/* constexpr.c -- constexpr int name = value */

#include <stdio.h>
#include <stdlib.h>

int main(int argc, [[maybe_unused]] char* argv[argc + 1]) {
    int constexpr LOWER = 0;
    int constexpr UPPER = 300;
    int constexpr STEP = 20;

    for (int fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP) {
        printf("%3d%10.2f\n", fahr, (5.0 / 9.0) * (fahr - 32));
    }
    return EXIT_SUCCESS;
}
```
