# 第 7 章 字符串

字符串是一种重要且实用的数据类型，大多数编程语言会实现为一种语言内建的类型。字符串通常用于表示文本，它构成了最终用户和程序交换的大部分数据。

在 C 语言中，字符串以串的形式化概念为模型。假设 $\Sigma$ 是字符的非空有穷集合，称为字符集。 $\Sigma$ 上的串是 $\Sigma$ 中任意有穷字符序列。

## 7.1 字符

我们用来交流的字符不能被数字洗头自然地理解，为了处理字符，数字洗头通过 **字符编码** 为每个字符分配唯一的整数值，这个值称为 **码点**。

在 [第 0 章 计算机基础#存储文本](./000-计算机基础.md#033-存储文本) 中介绍了用于组成字符串的各种字符集，包括 [ASCII](./000-计算机基础.md#ascii) 和 [Unicode](./000-计算机基础.md#unicode)，这里我们不再赘述

### 7.1.1 源字符集和执行字符集

最初在对 C 语言进行标准化时，尚没有普遍接受的字符编码，因此将语言设计为能够处理多种字符表示。每种 C 语言实现都会定义 **源字符集** 和 **执行字符集**

+ 源字符集：用于源代码书写
+ 执行字符集：用于在编译期解释字符和字符串

源字符集和执行字符集必须包含以下字符的编码：拉丁字母表的大小写字母、$10$ 个十进制数字、$29$ 个图形字符、空格、水平制表符、垂直制表符、馈页符和换行符。执行字符集还包括响铃字符、退格、回车和空字符

字符转换函数和字符分类函数在运行时根据函数调用时由 **区域设置** 决定的有效编码进行求值。区域设置定义了民族、文化和语言的当地惯例

### 7.1.2 字符类型

C 语言定义了多种用于表示字符数据的数据类型，在 [第 2 章 基本数据类型](./002-基本数据类型.md#212-使用字符变量) 中已经介绍了 `char` 类型

> [!tip]
>
> `char` 类型本质上就是整数类型，只是它通常只占 $8$ 位。这种字符类型通常称为 **窄字符**
>
> 通常我们使用 `int` 类型来表示，因为 `int` 类型能够存储更宽范围的值。我在读取字符时可能得到 `EOF`（一个负值），使用 `char` 可能无法表示。

C 语言还支持一种称为 **宽字符** 的类型，由 `wchar_t` 表示。本质上它也是某种整数，依赖于 C 语言的具体实现。大多数 C 语言实现将 `wchar_t` 实现为 $16$ 或 $32$ 位的无符号整数。实现可以有条件地将宏 `__STDC_ISO_10646__` 定义为 `yyyymmL` 地整数常量，指明用于表示特定标准对应地 Unicode 字符的 `wchar_t` 类型

从 C11 标准开始，引入 `char16_t` 和 `char32_t` 分别表示 $16$ 位和 $32$ 位的字符数据类型，它们定义在头文件 `<uchar.h>` 中，为 UTF-16 和 UTF-32 编码提供数据类型

C 语言定义了两个环境宏，指明如何对这些数据类型表示的字符进行编码。如果环境宏 `__STDC_UTF_16__` 的值为 $1$，那么 `char16_t` 类型的值就采用 UTF-16 编码。如果环境宏 `__STDC_UTF_32__` 的值为 $1$，那么 `char32_t` 类型的值就采用 UTF-32 编码

### 7.1.3 字符常量

C 语言允许指定字符常量，也称为 **字符字面量**，这是出现在一对 **单引号** 内的若干字符序列。在程序源代码中，可以使用字符字面量指定字符值。下表列出了能够在 C 语言源代码中出现的字符常量

| 前缀           | 类型         |
|:-------------|:-----------|
| `'A'`(无前缀)   | `int`      |
| `L'A'`       | `wchar_t`  |
| `u'A'`       | `char16_t` |
| `U'A'`       | `char32_t` |
| `u8'A'`(C23) | `char8_t`  |

> [!tip]
>
> 包含多个字符的字符常量，比如 `'ab'` 的值由实现定义。在执行字符集中不能表示为单个代码单元的源字符的值也是如此
>
> 从 C23 标准起，新增了 `char8_t` 类型，为 UTF-8 编码提供数据类型

### 7.1.4 转义序列

单引号(`'`)和反斜线(`\`)在 C 源代码中具有特殊含义，无法表示自身。想要表示单引号和反斜线可以使用转移字符序列。下表列出了 C 标准中定义的常见转移字符序列

| 字符     | 转移序列   |
|:-------|:-------|
| 单引号    | `\'`   |
| 双引号    | `\"`   |
| 问号     | `\?`   |
| 反斜线    | `\\`   |
| 响铃符    | `\a`   |
| 退格符    | `\b`   |
| 换页符    | `\f`   |
| 换行符    | `\n`   |
| 回车符    | `\r`   |
| 水平制表符  | `\t`   |
| 垂直制表符  | `\v`   |
| 八进制字符  | `\ooo` |
| 十六进制字符 | `\xhh` |

> [!tip]
>
> 八进制数字可以被纳入八进制转义序列，为字符常量构建 **单个字符**，或为宽字符常量构建单个宽字符。**八进制整数值指定了所需字符或宽字符的值**。反斜线后跟数字始终被解释为八进制值。如果表示单个字符，最多只支持 $3$ 八进制数。例如，可以将退格符（十进制 `8`）表示为八进制值 `\10` 或 `\010`
>
> 也可以将十六进制数字放在 `\x` 之后，为字符常量构建单个字符或宽字符。十六进制整数值就是所需的字符或宽字符的值。例如，可以将退格符表示为十六进制值 `\x8` 或`\x08`

### 7.1.5 字符转换

尽管国际文本越来越多地以 Unicode 编码，但是依赖于语言或国家地字符编码依然存在，因此有必要在这些编码之间进行转换。特别是，Windows 仍然在具有传统、有限字符编码地区域设置中运行。程序在执行 IO 时，经常需要在 Unicode 和传统编码方案之间进行转换

C 标准库提供了一些用于在窄代码单元(`char`)和宽代码单元(`wchar_t`)之间的转换函数

| 函数                   | 描述      |
|:---------------------|:--------|
| `mbtowc()/mbrtowc()` | 多字节到宽字符 |
| `wctomb()/wcrtomb()` | 宽字符到多字节 |

> [!tip]
>
> 转换函数需要存储数据以正确处理函数调用之间的一系列转换。其中不带 `r` 的函数称为不可重入版本，而带 `r` 的函数称为 **可重入** 版本。他们之间的区别在于状态是如何存储的。可重入版本需要一个额外的参数存储状态，这个参数的类型是 `mbstate_t` 类型的指针

从 C11 标准起，加入了下表列出的函数以解决标准转换函数可能造成的数据丢失

| 函数                      | 描述                  |
|:------------------------|:--------------------|
| `mbrtoc16()/c16rtomb()` | 多字节与`char16_t`之间的转换 |
| `mbrtoc32()/c32rtomb()` | 多字节与`char32_t`之间的转换 |

> [!tip]
>
> `mbrtoc16()/c16rtomb()` 在依赖于区域设置的字符编码，表示为 `char[]` 和存储在 `char16_t[]` 中的 UTF-16 数据之间的转换
>
> `mbrtoc32()/c32rtomb()` 在依赖于区域设置的字符编码，表示为 `char[]` 和存储在 `char32_t[]` 中的 UTF-32 数据之间的转换

下面的示例代码 [`ex07_01.c`](../src/ch07/ex07_01.c) 演示了使用 `mbrtoc16()` 函数将 UTF-8 字符串转换为 `char16_t` 字符串

```c title="ex07_01.c"
/**
 * ex07_01.c - 使用 `mbrtoc16()` 函数将 UTF-8 字符串转换为 `char16_t` 字符串
**/

#include <locale.h>
#include <uchar.h>
#include <stdio.h>
#include <stdlib.h>
#include <wchar.h>

#if __STDC_UTF_16__ != 1
#error "__STD_UTF_16__ NOT supported in this implementation"
#endif

int main()
{
    // 设置区域设置为支持 UTF-8 编码的中文环境
    setlocale(LC_ALL, "zh_CN.UTF-8");

    char input[] = u8"你好，世界！"; // UTF-8 编码的字符串

    const size_t input_size = sizeof(input);

    char16_t output[input_size]; // 输出缓冲区，大小足够容纳转换后的字符

    char *p_input = input;
    char *p_end = input + input_size;
    char16_t *p_output = output;
    size_t code;
    mbstate_t state = {0}; // 初始化转换状态
    puts(input);

    while ((code = mbrtoc16(p_output, p_input, p_end-p_input, &state)))
    {
        if (code == (size_t)-1)
        {
            break; // -1 检测到无效的代码单元序列
        }
        else if (code == (size_t)-2)
        {
            break;  // -2 检测到不完整的代码单元序列
        }
        else if (code == (size_t)-3)
        {
            p_output++; // -3 代理对偶（surrogate pair）中的高代理
        }
        else
        {
            p_output++; // 成功转换一个字符，移动输出指针
            p_input += code; // 移动输入指针，跳过已转换的字节
        }
    }
    size_t output_size = p_output - output; // 计算转换后的字符数量
    printf("Converted before: %zu bytes: [ ", input_size);
    for (size_t i = 0; i < input_size; ++i)
    {
        printf("%#x ", (unsigned char)input[i]); // 以十六进制格式输出 UTF-8 字节
    }
    puts("]");
    printf("Converted to %zu UTF-16 code units: [ ", output_size);
    for (size_t i = 0; i < output_size; ++i)
    {
        printf("%#x ", output[i]); // 以十六进制格式输出 UTF-16 代码单元
    }
    puts("]");
    return 0;
}
```

上述代码最终输出的结果为

```shell
你好，世界！
Converted before: 19 bytes: [ 0xe4 0xbd 0xa0 0xe5 0xa5 0xbd 0xef 0xbc 0x8c 0xe4 0xb8 0x96 0xe7 0x95 0x8c 0xef 0xbc 0x81 0 ]
Converted to 6 UTF-16 code units: [ 0x4f60 0x597d 0xff0c 0x4e16 0x754c 0xff01 ]
```

## 7.2 字符串

C 语言不支持原始字符串类型，并且可能永远不会支持。但是，C 语言将字符串实现为字符数组。C 语言有两种字符串：**窄字符串(`char[]`)**和 **宽字符串(`wchar_t`)**

窄字符串是 `char` 类型的数组，由包含 **终止空字符(`'\0'`)** 的连续字符序列组成。**字符串指针指向字符串的首个字符**。字符串的大小是其支撑数组所占的字节数。**字符串的长度是第一个空字符之前的代码单元数（字节数）**。如下图所示，字符串的大小为 $7$，长度为 $5$。支撑数组最后一个元素之外的元素不能访问。未初始化的数组元素不能读取

![1775054552448.png](https://fastly.jsdelivr.net/gh/duyupeng36/images@master/2026-04/1775054552468.png)

宽字符串是 `wchar_t` 类型的数组，由包含 **终止空字符(`L'\0'`)** 的连续宽字符序列组成。宽字符串指针指向宽字符串的首个宽字符。宽字符串的长度是第一个终止空宽字符之前的代码单元数。通常，`wchar_t` 至少占用 $2$ 字节，对于多字节对象的存储还涉及 **字节序** 问题

> [!tip]
>
> 对于多字节对象，每个字节在内存中如何排序？目前由两种公认的排序规则：大端法和小端法。如果一个 $n$ 位的整数，其位模式为 $[x_{n-1}, x_{n-2}, \cdots, x_{1}, x_{0}]$，其中 $x_{n-1}$ 是最高有效位，而 $x_{0}$ 是最低有效位。如果按照 $8$ 位进行分组，则 **最高有效字节** 包括 $[x_{n-1},x_{n-2},\cdots,x_{w-8}]$，而 **最低有效字节** 包括 $[x_{7}, x_{6}, \cdots,x_{0}]$。排列表示一个对象的字节有两个通用规则：
>
> + 某些机器选择在内存中按照从最低有效字节到最高有效字节的顺序存储对象，这种规则称为 **小端法**
>
> + 另一些机器则按照最高有效值字节到最低有效字节的顺序存储对象，这种规则称为 **大端法**
>
> 假设变量 `x` 的类型是 `int`，位于地址 `0x100` 处，它的十六进制值为 `0x01234567`。地址范围 $0x100 \sim 0x103$ 的字节顺序依赖于机器类型
>

### 7.2.1 字符串字面量

字符串字面量是由双引号中的一个或多个多字节字符表示的字符串常量。例如 `"hello, world!"`。可以使用各种前缀来声明不同字符类型的字面值常量

| 前缀           | 类型                       |
|:-------------|:-------------------------|
| `"ABC"`(无前缀) | `const char *`           |
| `L"ABC"`     | `const wchar_t *`        |
| `u8"ABC"`    | `const char8_t *`(C23起)  |
| `u"ABC"`     | `const char16_t *`(C11起) |
| `U"ABC"`     | `const char32_t *`(C11起) |

C 标准并没有强制要求实现对字符串字面量使用 ASCII。不过可以使用 `u8` 前缀强制字符串字面值使用 UTF-8 编码，如果字面值中所有的字符都是 ASCII 字符，则编译器会生成 ASCII 字符串字面量，即便实现通常以其他方式对字符串进行编码

> [!warning]
>
> 警告，字符串字面值本质上 `const char[]` 类型。由于数组与指针的密切连续，通常会将 `const char[]` 数组当 `const char *` 进行使用

如果需要对字符串进行改动，请声明 `T[]` 类型的数组，并使用字符串字面值进行初始化。这样就可以使用对字符串进行修改了。

```c
const char *message = "hello, world!";  // 不修改字符串的情形
char message[] = "hello, wiorld!"; // 需要修改字符串的情形
```

使用 `char[]` 声明字符串的一个优势就是获取数组大小可以使用 `sizeof` 运算符，这会在编译时进行计算。如果使用 `const char *` 声明字符串，就必须在运行时使用 `strlen()` 函数进行遍历计算字符串的长度，这会带来额外的运行时开销

### 7.3 字符串处理函数

在 C 语言中，处理字符串的方法有很多，我们介绍 C 标准库函数提供的处理方式。窄字符串处理函数和宽字符串处理函数分别定义在头文件 `<string.h>` 和 `<wchar.h>` 中。其中一部分函数不检查数组大小，从而导致缓冲区溢出。这是天生不安全的函数，需要谨慎使用，甚至是不使用

下表列出了常用的窄字符串函数和宽字符串函数

| 窄字符原型 | 宽字符原型 | 功能说明 |
|------------|------------|----------|
| `char *strcpy(char *dest, const char *src);` | `wchar_t *wcscpy(wchar_t *dest, const wchar_t *src);` | 字符串复制 |
| `char *strncpy(char *dest, const char *src, size_t n);` | `wchar_t *wcsncpy(wchar_t *dest, const wchar_t *src, size_t n);` | 截断，0 填充复制 |
| `void *memcpy(void *dest, const void *src, size_t n);` | `wchar_t *wmemcpy(wchar_t *dest, const wchar_t *src, size_t n);` | 复制指定数量的代码单元 |
| `void *memmove(void *dest, const void *src, size_t n);` | `wchar_t *wmemmove(wchar_t *dest, const wchar_t *src, size_t n);` | 移动指定数量（可能重叠）的代码单元 |
| `char *strcat(char *dest, const char *src);` | `wchar_t *wcscat(wchar_t *dest, const wchar_t *src);` | 拼接字符串 |
| `char *strncat(char *dest, const char *src, size_t n);` | `wchar_t *wcsncat(wchar_t *dest, const wchar_t *src, size_t n);` | 拼接字符串（会被截断） |
| `int strcmp(const char *s1, const char *s2);` | `int wcscmp(const wchar_t *s1, const wchar_t *s2);` | 比较字符串 |
| `int strncmp(const char *s1, const char *s2, size_t n);` | `int wcsncmp(const wchar_t *s1, const wchar_t *s2, size_t n);` | 比较字符串中指定个数的字符 |
| `char *strchr(const char *s, int c);` | `wchar_t *wcschr(const wchar_t *s, wchar_t c);` | 查找字符串中的某个字符 |
| `size_t strcspn(const char *s1, const char *s2);` | `size_t wcscspn(const wchar_t *s1, const wchar_t *s2);` | 检查字符串 str1 开头连续有多少个字符不在字符串 str2 中 |
| `char *strpbrk(const char *s1, const char *s2);` | `wchar_t *wcspbrk(const wchar_t *s1, const wchar_t *s2);` | 查找一组字符在字符串中首次出现的位置 |
| `char *strrchr(const char *s, int c);` | `wchar_t *wcsrchr(const wchar_t *s, wchar_t c);` | 查找某个字符在字符串中最后一次出现的位置 |
| `size_t strspn(const char *s1, const char *s2);` | `size_t wcsspn(const wchar_t *s1, const wchar_t *s2);` | 返回从字符串 str1 起始处起，完全由字符串 str2 所包含字符组成的子串长度 |
| `char *strstr(const char *haystack, const char *needle);` | `wchar_t *wcsstr(const wchar_t *haystack, const wchar_t *needle);` | 查找子串 |
| `char *strtok(char *str, const char *delim);` | `wchar_t *wcstok(wchar_t *str, const wchar_t *delim, wchar_t **ptr);` | 将字符串分割成一组子串（修改被分割的字符串） |
| `void *memchr(const void *s, int c, size_t n);` | `wchar_t *wmemchr(const wchar_t *s, wchar_t c, size_t n);` | 在内存中查找某个代码单元 |
| `size_t strlen(const char *s);` | `size_t wcslen(const wchar_t *s);` | 计算字符串长度 |
| `void *memset(void *s, int c, size_t n);` | `wchar_t *wmemset(wchar_t *s, wchar_t c, size_t n);` | 使用指定的代码单元填充内存 |

从 C11 引入了规范性边界检查接口，这类函数是可选的，旨在通过要求提供输出缓冲区的长度以及验证缓冲区的大小是否足以容纳这些函数的输出，来促进更安全、更可靠的编程。这类函数的命名方式是在原来不安全函数的末尾添加了 `_s` 表示安全函数(`safe`)

## 7.3 指针数组以及指向指针的指针

指针本身也是变量，所以可以像其他普通变量一样存储在数组中。我们通过一个示例程序 [`ex07_02.c`](../src/ch07/ex07_02.c) 来演示指针数组。这里，我们将排序一系列的长短不一的字符串。与排序整数不同的是，它们不能在单个运算中完成比较或移动操作。我们需要一个能够高效、方便地处理可变长度文本行的数据表示方法

我们引入 **指针数组** 处理这种问题。如果待排序的文本行首尾相连地存储在一个长字符数组中，**那么每个文本行可通过指向它的第一个字符的指针来访问**。这些指针本身可以存储在一个数组中。这样，将指向两个文本行的指针传递给函数 `strcmp` 就可实现对这两个文本行的比较。当交换次序颠倒的两个文本行时，**实际上交换的是指针数组中与这两个文本行相对应的指针**，而不是这两个文本行本身。如下图所示

![1775059592095.png](https://fastly.jsdelivr.net/gh/duyupeng36/images@master/2026-04/1775059592112.png)

> [!tip]
>
> 这种实现方法消除了因移动文本行本身所带来的复杂的存储管理和巨大的开销这两个孪生问题

```c title="ex07_02.c"
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int compare(const void *a, const void *b)
{
    return strcmp(*(const char **)a, *(const char **)b);
}

int main(void)
{

    char *lines[] = {
        "defghi",
        "jklmnopqrst",
        "abc",
    };

    qsort(lines, sizeof(lines) / sizeof(lines[0]), sizeof(lines[0]), compare);
    for (char **p = lines; p < lines + sizeof(lines) / sizeof(lines[0]); ++p) {
        printf("%s\n", *p);
    }
    return 0;
}
```

这个示例中，指针数组 `lines` 的声明是新出现的重要概念

```c
char *lines[] = {...};
```

数组 `lines` 是一个一维数组，其中每个元素都是一个 `char *` 类型的指针。也就是说，`lines[i]` 是一个字符指针，而 `*lines[i]` 是该指针指向的第 $i$ 个元素的首字符

由于 `lines` 本身是一个数组名，因此，可以按照 [第 5 章 指针与数组](./005-指针与数组.md#563-使用数组名的指针表示) 中介绍可以将数组名 `lines` 当指针使用。因此 `*lines` 就是数组 `lines` 的第一个元素 `lines[0]`。因此，数组 `lines` 被当作指针使用时，它是指向指针的指针。这就是这个示例程序出现的第二个重要概念

```c
char **p = lines; // 指向指针对象的指针
```

## 7.4 main() 函数的参数

在支持 C 语言的环境中，我们可以在程序开始执行时将命令行参数传递给程序。调用主函数 `main()` 时，它带有 $2$ 个参数。第一个参数习惯上命名为 `argc`，用于表示命令行参数个数。第二个参数习惯上名为 `argv`，是一个指向字符串的指针

按照 C 语言的约定，`argv[0]` 是启动该程序的程序名，因此 `argc` 的值至少为 $1$。此外，C 标准还要求 `argv[argc]` 的值必须是 `nullptr` 或者 `NULL`。下面的示例程序 [`ex07_03.c`](../src/ch07/ex07_03.c) 演示了一个简化版本 `echo` 程序

```c title="ex07_03.c"
#include <stdio.h>

int main(int argc, char *argv[])
{
    for (int i = 1; i < argc; ++i)
    {
        printf("%s%s", argv[i], (i < argc - 1) ? " " : "\n");
    }
    return 0;
}
```
